# プロジェクトディレクトリ構造

このドキュメントは、本プロジェクトの現在のディレクトリとファイルの構造、およびそれらの概要を示します。

## プロジェクトルート

```
.
├── .aicoding/       # AIエージェントの設定とルールを格納するディレクトリ
├── .github/         # GitHubのCI/CDワークフローやテンプレートを格納するディレクトリ
├── docs/            # プロジェクトのドキュメントを格納するディレクトリ
├── src/             # メインのソースコードを格納するディレクトリ
├── target/          # コンパイル成果物や中間ファイルを格納するディレクトリ
├── tmp/             # 一時的なファイルを格納するディレクトリ
├── .gitignore       # Gitが無視するファイルやディレクトリを指定するファイル
├── Cargo.lock       # プロジェクトの依存関係をロックするファイル
├── Cargo.toml       # プロジェクトのメタデータと依存関係を定義するファイル
├── clippy.toml      # Clippy（リンター）の設定ファイル
├── CONTRIBUTING-ja.md # プロジェクトへの貢献方法を記述したドキュメント（日本語）
├── README-ja.md     # プロジェクトの概要を記述したドキュメント（日本語）
├── README.md        # プロジェクトの概要を記述したドキュメント
└── rustfmt.toml     # rustfmt（フォーマッター）の設定ファイル
```

## `docs` ディレクトリ構造

`docs/` ディレクトリは、プロジェクトに関連するすべてのドキュメントを格納します。

```
docs/
├── 001-demands/             # プロジェクトの要求事項に関するドキュメント
├── 002-requirements/        # プロジェクトの要件定義に関するドキュメント
├── 003-designs/             # 設計関連のドキュメント
├── 004-adr/                 # アーキテクチャ決定記録（ADR）
├── 005-testing/             # テスト戦略やガイドラインに関するドキュメント
└── directory-structure.md   # このファイル。ディレクトリ構造の概要
```
